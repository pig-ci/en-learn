## 回報安全性漏洞 (Reporting a Vulnerability)

我們非常重視安全性問題。如果你發現任何安全性漏洞，請按照以下步驟回報：

### 聯絡方式

**請不要公開揭露漏洞**（不要在 GitHub Issues 或公開討論區回報）。

請將問題寄至：
- **Email**：`pig0911@shininglife-edu.com`

### 回報流程

1. **描述問題**：提供詳細的漏洞說明，包括：
   - 受影響的版本
   - 重現步驟
   - 潛在影響範圍
   - 任何相關的程式碼片段或截圖

2. **預期回應時間**：
   - 我們會在 **24-48 小時內** 確認收到您的回報
   - 我們會在 **7 天內** 提供初步評估
   - 我們會持續更新進度，直到問題解決

3. **處理方式**：
   - 我們會驗證漏洞是否存在
   - 我們會開發並測試修補程式
   - 我們會發布安全性更新
   - 我們會公開致謝（經您同意）

## 安全性最佳實踐

### 對開發者

- ✅ **環境變數**：敏感資訊（API Keys、密碼）務必使用環境變數
- ✅ **依賴更新**：定期更新依賴套件，修復已知漏洞
  ```bash
  npm audit
  npm audit fix
  ```
- ✅ **輸入驗證**：所有使用者輸入都需驗證與清理
- ✅ **HTTPS**：生產環境強制使用 HTTPS
- ✅ **最小權限原則**：只給予必要的權限

### 對使用者

- 🔒 請勿分享你的 API Key
- 🔒 定期備份你的學習資料（LocalStorage）
- 🔒 僅在信任的裝置上使用此應用

## 已知安全性措施

本專案已實施以下安全措施：

### 1. XSS 防護
- 使用 `DOMPurify` 過濾 AI 生成的 HTML 內容
- 避免使用 `dangerouslySetInnerHTML`（已透過 sanitize 處理）

### 2. API 安全
- Gemini API Key 僅在伺服器端使用
- 不在客戶端暴露任何敏感資訊

### 3. 依賴安全
```bash
# 定期掃描依賴
npm audit
# 或使用 Snyk
npx snyk test
```

### 4. 資料隱私
- 所有學習資料僅儲存在本地瀏覽器（LocalStorage）
- 不收集任何個人識別資訊
- 無第三方追蹤或分析
